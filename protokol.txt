Communication Protocol
Overview
Subsistem Magnetic Length Encoder menggunakan dua protokol komunikasi :
- RS485 - Komunikasi dengan master device/PLC
- Serial UART - Debug dan kalibrasi manual
RS485 Protocol
Frame Structure
┌─────────┬─────────┬────────┬──────┬─────────┬─────────┬──────────┬─────────┐
│ Header1 │ Header2 │ Length │ REQ  │ Address │ Command │   Data   │   CRC   │
│  0xD5   │  0xAA   │ 1 byte │ 0x88 │  0x06   │ 1 byte  │ Variable │ 2 bytes │
└─────────┴─────────┴────────┴──────┴─────────┴─────────┴──────────┴─────────┘
Frame Components
1. Header (2 bytes)
0xD5 0xAA - Fixed header untuk identifikasi frame
2. Length (1 byte)
Panjang data dari REQ sampai sebelum CRC
Length = 3 (REQ+ADDR+CMD) + Data_Length
3. REQ (1 byte)
0x88 - Request identifier
0x89 - Response identifier
4. Address (1 byte)
0x06 - Slave address (default)
Hanya frame dengan address 0x06 yang diproses
5. Command (1 byte)
0x00 - Length measurement request
0x02 - Control commands
0x03 - Error report (slave -> master saja, tidak pernah diminta master)
6. Data (Variable)
Tergantung command type
7. CRC (2 bytes)
Modbus CRC-16
Polynomial: 0xA001
Initial: 0xFFFF
Big-endian format (MSB first)
CRC-16 Modbus Calculation
uint16_t crc16_modbus(const uint8_t *data, uint16_t len) {
    uint16_t crc = 0xFFFF;

    for (uint16_t i = 0; i < len; i++) {
        crc ^= data[i];
        for (uint8_t j = 0; j < 8; j++) {
            if (crc & 0x0001)
                crc = (crc >> 1) ^ 0xA001;
            else
                crc >>= 1;
        }
    }
    return crc;
}
Command Set
CMD 0x00: Length Measurement
Request (Master to Slave)
D5 AA 04 88 06 00 00 [CRC_H] [CRC_L]
Fields:
- Length: 0x04 (4 bytes: REQ+ADDR+CMD+reserved)
- Command: 0x00
- Reserved: 0x00 (harus 0x00, selain itu dibalas Error)

Slave tidak langsung membalas request ini. Slave masuk mode "armed": membaca
AS5600 terus-menerus (dicetak ke Serial monitor untuk debug) sambil menunggu
klik tombol. Saat tombol diklik sekali, slave mengirim satu paket measurement
lalu kembali idle. Mode armed dibatalkan tanpa kirim paket jika: master
mengirim Cancel (CMD 0x02 SUBCMD 0x05), master mengirim Standby, atau timeout
60 detik tanpa tombol (slave kirim Error frame).

Response (Slave to Master) — dikirim saat tombol diklik
D5 AA 0C 89 06 00 [DIST_H][DIST_M][DIST_L] [ANGLE_H][ANGLE_M][ANGLE_L] [STAT_H][STAT_M][STAT_L] [CRC_H] [CRC_L]
Fields:
- Length : 0x0C (12 bytes: REQ+ADDR+CMD+3 byte dist+3 byte angle+3 byte status)
- DIST   : jarak/panjang untuk master ((hasil internal + 6.5 cm) x 100, dibulatkan 1 desimal) — 24-bit signed integer big-endian
- ANGLE  : sudut adjust (deg x 100, dibulatkan 1 desimal) — 24-bit signed integer big-endian
- STATUS : status tombol (x 100) — selalu 0x000064 karena slave hanya mengirim saat tombol diklik
Data Format:
int32_t length_cm_x100  = (int32_t)((DIST_H  << 16) | (DIST_M  << 8) | DIST_L);
float   length_cm       = length_cm_x100 / 100.0f;  // sudah termasuk offset +6.5 cm
int32_t angle_deg_x100  = (int32_t)((ANGLE_H << 16) | (ANGLE_M << 8) | ANGLE_L);
float   angle_deg       = angle_deg_x100 / 100.0f;
int32_t status_x100     = (int32_t)((STAT_H  << 16) | (STAT_M  << 8) | STAT_L);
Example:
Length internal = 8.3 cm
Length dikirim ke master = 14.8 cm
Encoded = 1480 (0x0005C8)
Bytes = [0x00, 0x05, 0xC8]

Syarat:
- State harus Operation. Jika Standby -> ERR_CHECKSUM.
- Reserved byte setelah CMD harus 0x00. Jika bukan -> ERR_CHECKSUM.
- AS5600 harus terkoneksi. Jika tidak -> ERR_SENSOR_NO_RESPONSE.
- Posisi pita saat request BEBAS: boleh "request dulu baru pita ditarik"
  maupun "pita ditarik dulu baru request" — dua-duanya valid. Nilai internal
  selalu posisi pita saat tombol diklik; DIST yang dikirim ke master ditambah
  offset 6.5 cm.

Semua kondisi gagal di atas dibalas lewat frame CMD 0x03 (lihat "CMD 0x03:
Error Reporting" di bawah), dan slave tidak masuk mode armed.

Selama mode armed aktif, jika sensor terputus saat polling, slave mengirim
ERR_SENSOR_NO_RESPONSE dan membatalkan mode armed. Jika 60 detik tanpa klik
tombol, slave mengirim ERR_TIMEOUT.

CMD 0x02: Control Commands
Request (Master to Slave)
D5 AA 04 88 06 02 [SUBCMD] [CRC_H] [CRC_L]
Fields:
- Length: 0x04 (4 bytes: REQ+ADDR+CMD+SUBCMD)
- Command: 0x02
- SUBCMD: control subcommand

SUBCMD      Command         Description
0x01        Standby         Enter low power mode (WiFi/BT off, CPU 80MHz)
0x02        Operation       Exit standby / ready
0x03        Tare            Set posisi AS5600 sekarang sebagai titik nol. Hanya diterima
                            bila pita sudah kembali home (jarak ke nol-sesi < TARE_HOME_TOL_CM,
                            default 5 cm) -> balas ACK Tare. Bila belum home -> ERR_TARE_INVALID
                            dan tare dibatalkan.
0x04        Restart         Reboot ESP32
0x05        Cancel          Batalkan mode armed (measurement/tare) tanpa kirim hasil

Response (Slave to Master) — ACK
D5 AA 05 89 06 02 00 [SUBCMD echo] [CRC_H] [CRC_L]
Fields:
- Length: 0x05 (5 bytes: REQ+ADDR+CMD+reserved+SUBCMD echo)
- Reserved: 0x00 (byte tetap setelah CMD, sebelum SUBCMD echo)
- SUBCMD echo: subcommand yang diminta master

Tare bisa slave-initiated: selain sebagai ACK atas perintah master, frame
Tare yang sama juga dikirim slave atas inisiatif user (tahan tombol 5 detik)
walaupun master tidak memintanya. Tare inisiatif user ini tunduk pada penjaga
home yang sama: bila pita belum home, slave mengirim ERR_TARE_INVALID (bukan
frame Tare) dan tidak melakukan tare.

SUBCMD selain 0x01-0x05 dibalas ERR_CHECKSUM lewat frame CMD 0x03 (lihat di
bawah), bukan lagi lewat CMD 0x02.

CMD 0x03: Error Reporting
Frame ini hanya dikirim slave -> master (slave-initiated), tidak pernah
diminta lewat request khusus dari master.

D5 AA 05 89 06 03 00 [CODE] [CRC_H] [CRC_L]
Fields:
- Length: 0x05 (5 bytes: REQ+ADDR+CMD+reserved+CODE)
- Reserved: 0x00 (byte tetap setelah CMD, sebelum CODE)
- CODE: kode error

CODE   Nama                     Kondisi pemicu
0x01   ERR_TIMEOUT              Mode armed measurement aktif tapi tombol
                                 tidak diklik dalam 60 detik
0x02   ERR_CHECKSUM             CRC mismatch pada frame ke address kita,
                                 atau data request tidak valid (reserved
                                 byte salah, SUBCMD/cmdType tidak dikenal,
                                 measurement diminta saat Standby)
0x03   ERR_SENSOR_NO_RESPONSE   AS5600 tidak terkoneksi saat master minta
                                 measurement, atau terputus saat armed
0x04   ERR_TARE_INVALID         Permintaan tare (master SUBCMD 0x03, atau tahan
                                 tombol 5 detik) sewaktu pita belum home (jarak
                                 ke nol-sesi >= TARE_HOME_TOL_CM, default 5 cm,
                                 lihat config.h). Tare dibatalkan.
